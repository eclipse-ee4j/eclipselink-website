Installing EclipseLink OSGi

Blah

Introduction

The Eclipse Plug-in Development Environment (PDE) provides tools to create, develop, test, debug, build and deploy Eclipse plug-ins, fragments, features, update sites and RCP products. PDE also provides comprehensive OSGi tooling. For more information about PDE, see "PDE" at http://www.eclipse.org/pde/.

REWRITE THIS::::::::::::::::::To use PDE, PDE must be installed on your computer. PDE ships with the Eclipse SDK and can be downloaded from the Eclipse project downloads page http://www.eclipse.org/downloads/. For installation instructions, see "Eclipse Installation" at http://wiki.eclipse.org/Eclipse/Installation.

If you already have an installation of Eclipse that does not include PDE (for example, Eclipde IDE for Java Developers), you can install it using the Install New Software wizard. Filter the results for "Eclipse plug-in development environment." PDE is listed as "Eclipse plug-in development environment" under the "General Purpose Tools" category.

EclipseLink OSGi bundles are made available both as a zip and via an Eclipse P2 update site. See the following for download and installation instructions.

  • Alternative 1: Install EclipseLink OSGi from the Zip File

  • Alternative 2: Install EclipseLink OSGi in PDE from an Eclipse P2 Update Site

Alternative 1: Install EclipseLink OSGi from the Zip File

To download and install the zip file distribution, do the following:

  1. Download the EclispeLink OSGI bundle zip file from http://www.eclipse.org/eclipselink/downloads/.

  2. Unpack the downloaded zip file in the desired installation directory. You can do either of the following:

    • Add the installation directory to a target platform.

    • If you are developing in Eclipse with PDE, you can place the downloaded bundles into your <ECLIPSE_HOME>\dropins folder. However, the approach described in Alternative 2: Install EclipseLink OSGi in PDE from an Eclipse P2 Update Site is recommended.

Alternative 2: Install EclipseLink OSGi in PDE from an Eclipse P2 Update Site

To Install EclipseLink OSGi in PDE from an Eclipse P2 update site, do the following:

  1. Start the Eclipse IDE.

  2. From the Open 'Window>Preferences' and navigate to 'Plug-in Development>Target Platform'.

  3. Select 'Add..' to create a new target platform definition.

  4. Select 'Nothing' to start with an empty target and then click 'Next'.

  5. On the 'Locations' tab select 'Add..'.

  6. Select 'Software Site' and then click 'Next'.

  7. Choose the Helios update site from the 'Work with:' drop down of P2 repositories.

  8. Once the feature list has been populated, from the 'EclipseRT Target Platform Components' category check 'EclipseLink Target Components' and click 'Finish'.

  9. Change the name of your new target platform to reflect its contents, e.g., 'EclipseLink 2.1'.

  10. Finally, select your newly defined target as your default and click 'Ok'.

In the Plug-ins view of the Plug-in Development perspective you will see all of the bundles in your target platform. Note that bundles EclipseLink requires have also been downloaded and installed by P2, e.g., Equinox (org.eclipse.osgi).

EclipseLink OSGi Bundles

The following tables list the bundles included with EclipseLink OSGi.


Note:

The names of the bundle files include suffixes with a release and build number. Those numbers vary depending upon the release you download.

Table 1-1 EclipseLink Core Functionality

Bundle Description

org.eclipse.persistence.core

This bundle provides EclipseLink's core library. All other EclipseLink bundles (JPA, MOXy, SDO) are dependent on this bundle.

org.eclipse.persistence.asm

This bundle provides EclipseLink's version of the ASM library used for byte-code weaving in the core, JPA, SDO and MOXy bundles. It is required when using any of those bundles.

org.eclipse.persistence.antlr

This bundle provides EclipseLink's version of the ANTLR library used for JPQL parsing in the JPA and core bundles. It is required when using JPA.

org.eclipse.persistence.oracle

This bundle provides support for extended Oracle database features (Timestamp extensions, Spatial types, Oracle XML Parser, XDB, Advanced Queueing, XDK, Oracle LOBs).

javax.activation

javax.jms

javax.mail

javax.resource

javax.servlet

javax.transaction

javax.xml.rpc

javax.xml.soap

javax.xml.stream

javax.xml.ws

These bundles contain classes from Java specification. They have been converted to bundle format so they can be used in OSGi. Depending on the OSGi environment whether the running OSGi Environment EclipseLink contains those versions of the libraries listed, these bundles may or may not be necessary.

Table 1-2 EclipseLink JPA

Bundle Description

javax.persistence

This bundle contains versions of the javax.persistence and javax.persistence.spi classes that have been updated to be OSGi friendly. OSGi users must use this bundle for performing OSGi-based JPA.

org.eclipse.persistence.jpa

This bundle supports JPA functionality along with Extended JPA support using EclipseLink's annotations, PU properties, and query hints. The bundle also supports partial and complete EclipseLink-ORM.XML extensions of JPA.

org.eclipse.persistence.jpa.osgi

This bundle provides support for using JPA in OSGi

org.eclipse.persistence.jpa.equinox

org.eclipse.persistence.jpa.equinox.weaving

These bundles implement byte code weaving support when running on Equinox, the Eclipse implementation of the OSGi R4 core framework specification, including a set of bundles that implement various optional OSGi services and other infrastructure for running OSGi-based systems. For information about Equinox, see http://www.eclipse.org/equinox.

Table 1-3 EclipseLink MOXy

Bundle Description

org.eclipse.persistence.moxy

This bundle supports JAXB 2.0 using annotations and XML mapping file.

javax.xml.bind

This bundle contains the specification classes and interfaces augmented with support for discovering JAXB provider services.

Table 1-4 EclipseLink SDO

Bundle Description

org.eclipse.persistence.sdo

This bundle supports Eclipse persistence SDO 2.1 classes.

commonj.sdo

This bundle provides common SDO 2.1 classes.
